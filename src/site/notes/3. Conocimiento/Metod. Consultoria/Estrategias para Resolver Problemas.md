| Estrategia                             | Explicación                                                                                                                                                                                |
| -------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Datos del Problema**                 | Identificar cuales son los datos del problema y cuales podrían faltar para la solución.                                                                                                    |
| **Trabajar hacia atrás**               | Definir el objetivo final y desandar los pasos necesarios. Usar el fraseo: Para lograr esto, debo haber logrado esto, esto y esto. Y asi sucesivamente hasta llegar a un inicio abordable. |
| **Invertir el problema**               | Resolver el problema inverso y después aplicar la solución contraria obtenida al problema original.Charly Munger (QUEPD) pregonaba mucho esta.                                             |
| **Buscar problema similar**            | Analizar cómo se resolvió una situación parecida en otro ámbito y extrapolarla.                                                                                                            |
| **Resolver versión simplificada**      | Buscar un problema similar al que se tiene entre manos pero diferente de algún modo que lo haga mas fácil resolver. O bien abordar primero una versión menos compleja del problema.        |
| **Dividir en subproblemas**            | Fragmentar el desafío en partes manejables e independientes.                                                                                                                               |
| **Buscar un patrón**                   | Identificar recurrencias o estructuras que guíen la solución.                                                                                                                              |
| **Realizar un dibujo**                 | Representar gráficamente el problema para clarificar su esquema.                                                                                                                           |
| **Análisis de metas**                  | Comparar el estado actual con la meta y reducir el mayor desajuste.                                                                                                                        |
| **Primeros principios**                | Descomponer el problema hasta sus verdades fundamentales y recomponer la solución. Elon Musk pregona mucho esta. Partir de una hoja en blanco.                                             |
| **Relajar o restringir restricciones** | Quitar o añadir límites para descubrir nuevas vías de resolución.                                                                                                                          |
| **Probar casos extremos**              | Evaluar la solución bajo condiciones límite para detectar fallos.                                                                                                                          |
| **Abstraer y generalizar**             | Reformular el problema en términos más amplios para aprovechar modelos conocidos.                                                                                                          |
| **Especializar**                       | Imponer supuestos adicionales para simplificar y acotar el problema.                                                                                                                       |
| **Verificar suposiciones**             | Enumerar y cuestionar cada supuesto para evitar cimientos erróneos.                                                                                                                        |
| **Lluvia de ideas y poda**             | Generar ideas sin filtro y luego seleccionar las más prometedoras.                                                                                                                         |
| **Hipotetizar y experimentar**         | Formular una hipótesis y testearla rápidamente para obtener retroalimentación.                                                                                                             |
